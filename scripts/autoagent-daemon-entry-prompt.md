/loop 24h AutoAgent 일일 사이클 보조 실행. 다음 절차를 순서대로 처리하라. (0a) Cron 등록 무조건 선행 (idempotent, deadlock-proof): 먼저 `CronList` Tool 호출로 현재 세션의 스케줄 목록을 조회한다. 결과 prompt 필드에 "AutoAgent 일일 사이클" 문자열을 포함하는 항목이 이미 존재하면 본 단계 등록은 스킵하고 (0b)로 진행한다(중복 등록 방지). 존재하지 않으면 `CronCreate` Tool 을 다음 인자로 호출한다: `cron="0 3 * * *"`, `recurring=true`, `prompt`=본 entry-prompt 파일의 (1)~(4) 본문 전체((0a)/(0b) 단계 자체는 제외). CronCreate 가 `Scheduled <8-hex-id>` 를 응답하면 healthcheck Step 5 (`Scheduled [a-f0-9]{8}` grep) 가 만족된다. **Ordering invariant**: Cron registration (0a) 는 어떤 quota 상태에서도 무조건 첫 LLM 호출로 수행한다. Quota wait (0b) 는 본 사이클 body 만 중단하며 schedule 등록 자체는 차단하지 않는다 — 이전 (0a) quota gate → (0b) CronCreate 순서가 quota 도달 시 CronCreate 영구 skip 으로 데드락을 유발했던 구조를 reverse 하여 데드락이 구조적으로 불가능하다. CronCreate 자체가 실패하면(매우 드물지만 발생 가능) Bash 로 `printf '[%s] [autoagent-cron-fail] CronCreate failed: <응답 1줄>\n' "$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> /tmp/autoagent-daemon-loop.log` 실행 후 본 사이클 종료(silent fallback 금지). (0b) Quota 사후 검사: `Bash` 으로 `tmux capture-pane -t claude-autoagent-daemon -p -S -10 | grep -E "Limit reached|Usage ⚠ Limit" | head -1` 평가. 결과가 비어있으면(quota 정상) (1) 진행. 결과가 quota 한도 도달이면(예: "Usage ⚠ Limit reached (resets 2h 10m)") `Bash` 으로 `printf '[%s] [autoagent-quota-wait] Limit reached, retry after next cron fire\n' "$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> /tmp/autoagent-daemon-loop.log` 실행 후 본 사이클 즉시 종료(LLM 추가 호출 금지). 다음 cron fire (다음날 03:00 KST, (0a) 에서 이미 등록 완료) 또는 launchd daily-restart (KST 03:30) 에서 재시도. (1) Phase 2 인프라 가드: bash로 `test -x ~/.claude/autoagent/daemon-cycle.sh` 평가. 없거나 실행권한 미부여이면 Bash로 `printf '[%s] Phase 2 미배포 — 대기 중\n' "$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> /tmp/autoagent-daemon-loop.log`만 실행하고 본 사이클 종료 (LLM 추가 호출 금지).
