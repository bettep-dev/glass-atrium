Read-only replay against `core.outcomes`. This note is the sole deliverable: no
production table was written and no code was changed. Its existence is the gate
T2 depends on ("T2 shall not begin until this note exists"), and it is the
reference the T21 `grader_verdict` delta table is produced against.

## Replay provenance (the three specifics the contract requires)

- **Pinned predicate commit**: `a627de73665e9708e3fd9275b6c0f605b9c506f5`
  (`a627de7`, branch `feature/harness-remediation-290`). The CTM admission rate
  is not a queryable rate — the lesson store is a current-state snapshot with no
  denominator and no admit timestamps — so it is obtained by replaying the live
  bucket-routing function `classify_lesson_bucket` (in `hooks/learning-aggregator.py`
  at this commit) over the in-window rows. Before/after runs must pin the same
  predicate version or the delta measures the predicate change, not the data.
- **Window column**: `record_ts` (the record timestamp / data-progress point the
  learning-aggregator watermarks on, `outcomes_ts_idx`). The table also carries
  `inserted_at` (DB-write clock, `DEFAULT CURRENT_TIMESTAMP`); the two are **not**
  interchangeable, and the baseline window is the record clock, not the insert clock.
- **NULL grader-verdict exclusion**: the replay predicate is `grader_verdict IS NOT NULL`.
  At this DB state it excludes **0** rows (the legacy NULL-verdict rows the plan
  noted as "confirmed present" are not present in the live table today); the
  exclusion is nonetheless applied so before/after runs share one predicate.

grader to the bug-fix and feature arms only.
